# Skill: UI Polish

Purpose:

Refine existing UI without redesigning it.

Improve:

- spacing
- hierarchy
- consistency
- responsiveness
- empty states
- subtle animations
- visual balance

Preserve:

- Spottr personality
- existing layout
- navigation structure
- component patterns

Do not redesign unless asked.

---

## Spottr Feel

UI should feel:

- warm
- approachable
- playful but trustworthy
- calm
- human
- never corporate
- never intimidating

---

## Review Checklist

### Layout

Check:

- spacing
- alignment
- empty space
- padding consistency

### Hierarchy

Check:

- what stands out first
- CTA clarity
- scanability
- visual fatigue

### Components

Check:

- card consistency
- button consistency
- icon sizing
- input styling

### Navigation

Check:

- safe area handling
- gesture bar overlap
- active states
- touch targets

### Empty States

Check:

- intentionality
- warmth
- subtle motion

Prefer:

- pulse dots
- radar ripple
- floating Spottr pins

Avoid:

- emojis
- mascots
- flashy motion

---

## Implementation Style

Prefer:

- small styling changes
- reusable improvements
- subtle animations

Avoid:

- major rewrites
- dependency bloat

---

## Output Format

1. UI audit
2. What feels off
3. Suggested improvements
4. Files affected
5. Code changes
6. Why this improves UX
