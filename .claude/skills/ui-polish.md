# UI Polish Skill

Purpose:
Review and refine existing UI without redesigning it.

This skill is for improving:
- spacing
- hierarchy
- typography
- visual balance
- consistency
- polish
- premium feel
- responsiveness

Do NOT redesign unless explicitly asked.

---

## Core Behavior

When reviewing UI:

1. Audit what feels visually off
2. Identify friction or inconsistency
3. Recommend improvements
4. Implement only targeted polish
5. Preserve existing design language

Always explain:
- what feels off
- why it feels off
- how the proposed change improves it

---

## Preserve Existing Identity

Respect the current product.

Do not change:
- app personality
- color system
- layout structure
- navigation model
- interaction patterns

Only refine them.

Avoid:
- random redesigns
- unnecessary new components
- trend-chasing UI
- adding visual clutter

---

## Review Criteria

### 1. Spacing & Layout
Check for:
- excessive empty space
- cramped sections
- uneven margins
- inconsistent padding
- alignment issues
- poor visual balance

### 2. Visual Hierarchy
Check:
- what draws attention first
- whether important actions stand out
- whether cards look too similar
- whether users can scan quickly

Improve with:
- spacing
- size
- contrast
- weight
- surface treatment

### 3. Typography
Check:
- header sizing
- subtitle contrast
- readability
- consistency
- line height

Prefer:
- fewer type sizes
- stronger hierarchy
- calm readability

### 4. Component Consistency
Check:
- border radius consistency
- card treatment consistency
- icon sizing
- button styling
- input field consistency

### 5. Navigation Polish
Check:
- active states
- spacing
- touch targets
- safe area issues
- tab bar weight
- gesture bar overlap

### 6. Empty States
Check:
- intentionality
- visual balance
- clarity

Avoid empty screens that feel unfinished.

### 7. Brand Feel
Ask:
Does this feel like the product?

For Spottr:
- warm and approachable
- playful but trustworthy
- human and honest
- a little self-aware
- never corporate, never cold
- not like a dating app
- not like a generic gym app

### 8. Responsiveness
Check:
- mobile spacing
- small device layout issues
- safe area handling
- bottom navigation overlap

---

## Implementation Style

Prefer:
- small styling changes
- spacing adjustments
- better hierarchy
- subtle animations
- component refinements

Avoid:
- major rewrites
- dependency bloat

---

## Output Format

Always respond with:

1. UI audit
2. What feels off
3. Suggested improvements
4. Files affected
5. Styling changes
6. Code changes
7. Why this improves UX
