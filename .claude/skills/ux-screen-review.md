# Skill: UX Screen Review

Use this when asked to review or improve any Spottr screen.

## Context
- App: Spottr — gym connection app, Metro Manila
- Stack: Expo React Native, TypeScript, React Native StyleSheet
- No Tailwind or NativeWind

## Product Tone
- calm, premium, minimal
- consent-first, socially reassuring
- not a dating app, not a chat app

## Output Format
1. UX critique — what's wrong and why
2. Layout recommendations — specific changes
3. Updated component code — full updated file
4. Explain why each change improves UX

## Rules
- Do not redesign the whole app
- Only improve the requested screen
- Preserve dark premium aesthetic
- Keep mobile-first design
- Preserve Spottr branding and tone
